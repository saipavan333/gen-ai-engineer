Watch a prompt get reliable

A prompt isn't magic words — it's a set of components, each removing a specific way the model can disappoint you. Watch a bare instruction become a production prompt, and watch three sample runs stop disagreeing.

How to use this lab

Press ▶ Play to add the components one at a time, or Step to pause and read what each one does. Watch the right-hand panel: the three runs start as different prose and end as one identical, parseable answer.

Make it yours

1. At the + Format stage the outputs look like JSON but the verdict still says they'll break a parser. Look closely at the three — what's different between them?
2. Which single component moved the runs from 'three different answers' to 'accurate but unstructured'? Which one made them identical?

▸ Show reasoning

The three JSON-ish outputs differ in tiny ways — a space inside a value, single quotes instead of double, an extra key. Each is 'almost JSON', and almost-JSON throws an exception the moment you call a strict parser on it. That's why format alone isn't enough: the guardrail ("never invent fields", "if unsure use other") is what finally pins every run to the same valid shape. Context fixed the accuracy; the guardrail fixed the reliability.

What you just saw
  • A prompt as components, not a magic sentence.
  • Each component removing one specific class of failure.
  • The difference between almost-parseable and parseable.
