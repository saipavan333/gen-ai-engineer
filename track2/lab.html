Lab: Build a prompt engine

Prompts stop being strings you paste and start being code you can test. You’ll build a template engine, a few-shot builder, and the parse-and-validate layer that turns a model’s chattiness into data your program can trust.

How to use this lab

Run the cells in order — they share memory. All plain Python, running right here.

Step 1 — Templates, not strings

Hard-coded prompts don’t scale. A template with named fields lets you generate thousands of consistent prompts from data — and change the wording in one place.

step1_template.py
TEMPLATE = ("You are a concise support agent.\n"
            "Answer in {sentences} sentences, in a {tone} tone.\n\n"
            "Question: {question}")

def build_prompt(template, **fields):
    return template.format(**fields)

print(build_prompt(TEMPLATE, sentences=2, tone="friendly",
                   question="How long do refunds take?"))

Step 2 — Teach by example (few-shot)

Instructions describe the task; examples demonstrate it. For classification, two or three examples usually beat a paragraph of rules.

step2_fewshot.py
EXAMPLES = [
    ("Where is my order?",        "shipping"),
    ("I want my money back",     "refund"),
    ("The app crashes on login", "bug"),
]

def few_shot_prompt(question):
    shots = "\n".join("Ticket: " + t + "\nLabel: " + l for t, l in EXAMPLES)
    return ("Classify the support ticket. Reply with one label only.\n\n"
            + shots + "\n\nTicket: " + question + "\nLabel:")

print(few_shot_prompt("My card was charged twice"))

Step 3 — Ask for JSON, then survive the reply

Asking for JSON is easy; getting clean JSON every time is not. Models add friendly preambles. Build the salvage-and-parse layer that assumes it will misbehave.

step3_parse.py
import json, re

def fake_model(sloppy=False):
    if sloppy:
        return 'Sure! Here you go: {"intent": "refund", "urgency": "high"}'
    return '{"intent": "refund", "urgency": "high"}'

def parse_json(text):
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        m = re.search(r"\{.*\}", text, re.S)    # salvage the object
        if m:
            try:
                return json.loads(m.group(0))
            except json.JSONDecodeError:
                pass
    return None

print("clean :", parse_json(fake_model()))
print("sloppy:", parse_json(fake_model(sloppy=True)))
print("junk  :", parse_json("no json at all"))

Step 4 — Validate before you trust

Parsing proves it’s JSON. Validation proves it’s the JSON you asked for. This is the layer that stops bad data reaching your database.

step4_validate.py
ALLOWED_URGENCY = {"low", "high"}

def validate(obj):
    if not isinstance(obj, dict):
        return False, "not an object"
    for field in ("intent", "urgency"):
        if field not in obj:
            return False, "missing field: " + field
        if not isinstance(obj[field], str):
            return False, "wrong type: " + field
    if obj["urgency"] not in ALLOWED_URGENCY:
        return False, "urgency must be low or high"
    return True, "ok"

for raw in ['{"intent":"refund","urgency":"high"}',
            '{"intent":"refund"}',
            '{"intent":"refund","urgency":"URGENT"}',
            'not json']:
    obj = parse_json(raw)
    result = validate(obj) if obj else (False, "unparseable")
    print(result, "|", raw)
What you built
  • A template engine — prompts generated from data, edited in one place.
  • A few-shot builder that teaches by demonstration.
  • A tolerant parser that salvages JSON from chatty replies.
  • A validator that rejects the right-shaped-but-wrong answers.

Parse and validate is the pattern behind every reliable LLM feature. Structure in, structure out, nothing trusted in between.

▸ Show reasoning

Retry once (truncation is often transient), then fall back to a safe default and log it. Failing loudly to the user is right only when a wrong value is worse than no value — for a support classifier, defaulting to “needs human review” is far safer than guessing “low urgency.” Match the fallback to the cost of being wrong.
