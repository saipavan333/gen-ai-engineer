Track 7 · Visual Lab

Watch an attack meet your defences

There is no prompt that makes a system safe. There are only layers, each one imperfect. Watch a real injection payload travel through five of them — then switch layers off and find out which ones were actually doing the work.

How to use this lab

Play it with everything on. Then turn layers off with the buttons and run it again. The question to hold in mind is not “is it blocked?” but “what survives, and can I live with it?”

Make it yours

1. Turn off everything except Least privilege. Is the system still safe? Now turn off everything except Input filter. Compare.
2. Which layers depend on the model behaving, and which hold regardless of what the model does? Which kind would you rather bet a refund budget on?

▸ Show reasoning

1. Least privilege alone still contains the damage, because a capped tool cannot be talked into exceeding its cap. Input filtering alone fails — it only catches phrasings someone already thought of, and an attacker just rewrites the sentence. Same number of layers, completely different worst case.

2. Input filtering and instruction hierarchy are persuasion-based: they work by convincing the model, and anything that convinces can be out-convinced. Least privilege, output validation and human approval are structural: they hold even when the model is fully compromised. Bet the money on the structural ones and treat the rest as friction that buys you time.

What you just saw
  • An injection arriving as ordinary retrieved data.
  • Layers that work by persuading the model — and their limits.
  • Layers that hold structurally, whatever the model decides.
  • The real security question: what survives everything.
